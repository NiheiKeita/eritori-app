# エリトリ 実装仕様書 v1.0（Flutter）

> 原本仕様書。実装はこのドキュメントを正とする。実装計画・アーキテクチャ判断は [plan.md](./plan.md) を参照。

## 0. このゲームを一言で

エリマキトカゲの「襟」を指でなぞって切り取るカジュアルゲーム。
回ってくる顔をかわしながら、顔に触れずに襟を大きくえぐり取る。良い襟を集めてエリボードに展示し、画像でSNS共有する。

縦持ち専用。録画・スクショ映えを最優先する。

## 1. 技術前提

- Flutter。ゲーム描画は `CustomPainter` + `GestureDetector` ベースで自前実装。物理エンジン不要。
- 状態管理は既存プロジェクト方針（InheritedWidget `AppScope` によるDI + Container/Presentation/Controller パターン）。
- 永続化はローカルのみ。サーバー通信なし。共有はOS共有シート。
- ドメインロジック（スコア計算・判定）を UI から分離し、純粋関数として単体テスト可能にする。コンストラクタインジェクションでサービスを注入する。

### 推奨パッケージ

- `path_provider` … 襟画像とJSONの保存先
- `share_plus` … OS共有シート
- `RenderRepaintBoundary` … エリボードの画像化（標準APIで実装）
- `audioplayers` … 効果音
- 軽量アニメは `AnimationController`、パーティクルは自前 CustomPainter

## 3. データモデル

### Eri（襟）
- `id`(UUID), `stageId`, `stageName`, `score`(0..10000), `captureRate`(0..1), `quality`(D/C/B/A/S), `acquiredAt`, `imagePath`, `isPersonalBest`, `location`(chest/board), `boardSlotIndex?`
- 重要：襟の実体は1つ。`location` が chest か board のどちらかを指す（「移動」モデル）。コピーはしない。

### Stage
- `id`, `name`, `theme`(ネオン配色), `lizardAsset`, `eriAsset`, `shape`, `rotates`, `rotationSpeed`(deg/sec), `clearScore`, `unlocked`

### Quality
- `enum Quality { D, C, B, A, S }` — `captureRate` から決定。

## 4. 永続化（ローカル）

- 襟メタデータは JSON 配列（`eris.json`）。
- 襟画像は PNG ファイルとして documents ディレクトリに保存し、パスをメタに持たせる。長辺512px程度にリサイズ。
- 進捗（解放レベル・各ステージ自己ベスト）は `progress.json`。
- アプリ削除でデータが消えることは許容（MVPバックアップなし）。

```

  └─ 2回目以降 → ホーム
ホーム（=レベル選択 / 下タブ起点）
  ├─ [タブ] 宝箱  ├─ [タブ] エリボード

ゲーム画面

  └─ 顔に触れる / 不成立 → 失敗画面
リザルト（成功）
  ├─ リトライ → ゲーム  ├─ レベル選択 → ホーム  ├─ 破棄 → ホーム
  └─ 宝箱に保管 → 満杯? 空き→宝箱 / 満杯→整理画面→入れ替え確定→宝箱

エリボード ├─ 襟タップ → 詳細ポップアップ（その場）└─ SNS共有 → OS共有シート
```

## 6. コアゲームロジック（最重要）

### 6.1 座標系の原則
- すべての判定・面積計算は襟スプライトのローカル座標で行う。
- 毎フレーム、指先の画面座標に「襟スプライトの変換行列の逆行列」をかけてローカル座標へ変換し、Stroke に追加。
- 回転レベルでも静止レベルと同一コードで判定。

### 6.2 なぞり〜確定フロー
- onPanStart: stroke.clear(); stroke.add(toLocal(p))
- onPanUpdate: local = toLocal(p, currentSpriteTransform)
  - 失敗判定（最優先）: 今の指先が本体(非透過)に触れたら即失敗
  - 自己交差判定: 新線分が既存線分と交わったら確定(loop)
  - else stroke.add(local)
- onPanEnd（交差せず離した）: FAIL（不成立）
- 失敗判定は「今の指先」だけを本体マスクと照合。描き終えた線全体は照合しない。
- 8の字は最初の交差で即確定し、その交差で閉じたループを対象。

### 6.3 本体マスク判定
- トカゲ本体PNGの透過情報から当たり判定マスク。α > 閾値（例 0.5）を本体とする。
- 起動時に本体PNGを ui.Image → ピクセルバッファ化、ローカル座標→ピクセル参照で isOpaqueAt を O(1)。
- 回転レベルでもローカル座標で本体は静止 → マスク1枚で済む。

### 6.4 面積・スコア計算
1. ループで囲まれた領域を求める
2. **捕捉対象は「顔（中心）を含まない側」**＝なぞりで切り離した外側の襟。
   ループが顔中心を内側に含む場合は補集合(全体−内側)を捕捉領域とする。
3. 捕捉領域内の襟(eriマスク非透過)ピクセル数 = capturedEriPixels
4. 襟全体の非透過ピクセル総数 = totalEriPixels（ステージ定数として事前計算）
5. captureRate = capturedEriPixels / totalEriPixels
6. score = round(captureRate * 10000)
7. quality = qualityFromRate(captureRate)

### 6.2b 早期確定の防止（誤検出対策）
- なぞり点は前点から `minPointDist`(ローカル) 以上離れた時だけ追加（間引き）。密な極小線分による
  不安定を防ぐ。
- 自己交差は**最新線分 vs 過去線分の厳密交差のみ**で判定（端点接触・共線は交差扱いしない）。
  隣接線分は `minIntersectionGap` 本ぶん除外。
- 閉じ判定は `minPointsBeforeClose` 点たまってから開始。
- できるループの面積が `minLoopArea`(ローカル) 未満なら退化ループとして無視し確定しない。
- ※以前の「線の太さ＝距離トレランス」方式は近接セグメントの誤検出で早期確定を招いたため撤去。
- ピクセルはローカル座標（元PNG解像度）で数える → 端末差なし。
- ループ内走査はバウンディングボックス内 + 偶奇規則(point-in-polygon)。判定用に縮小(256〜512px)可。

### 6.5 切り取り画像の生成
- 確定後、ループ多角形でマスクした襟領域を切り出して PNG 化（透過背景）。`Eri.imagePath` に保存。
- 切り口はアンチエイリアスで「スパッと切れた」断面に。

## 7. ゲームバランス（config・数値外出し）

### 7.1 クリア基準（正規化スコア＝捕捉率）
| Lv | テーマ | 特徴 | 回転 | クリア基準 |
|----|------|------|----|-----------|
| 1 | 草原 | 静止・シンプル | × | 7,000 |
| 2 | 海 | 耳付き | × | 7,500 |
| 3 | 火山 | 顎付き | × | 8,000 |
| 4 | 雷 | Lv1形状・回転 | ○ | 7,500 |

### 7.2 品質ランク閾値
| ランク | 捕捉率 |
|------|------|
| S | 95%以上 |
| A | 88〜95% |
| B | 78〜88% |
| C | 65〜78% |
| D | 65%未満 |

### 7.3 回転速度
- Lv4 の rotationSpeed(deg/sec) も config。暫定は中速。

### 7.4 容量
- 宝箱初期 5／エリボード初期 2。拡張はレベルアップ報酬（Lv5:+5, Lv10:+5, Lv20:+10）。config。

## 8. 画面別 詳細仕様（要点）
- ホーム/レベル選択: 下タブ3つ起点。未解放はロック+基準表示。自己ベスト%とサムネ。
- ゲーム: 縦中央にトカゲ+襟。なぞり中スコア非表示。離した瞬間にドン。線は襟と一緒に回る。
- リザルト: スコア/捕捉率%/品質/自己ベスト更新/プレビュー。保管/破棄/リトライ/レベル選択。
- 失敗: 赤系・別トーン。変顔リアクション。リトライ/レベル選択のみ。
- 宝箱: グリッド。襟選択→ボードへ移動。満杯表示。
- エリボード: 1枚の大きなボードに襟画像を自由配置（ドラッグ移動・ピンチ拡縮・重ね可）。
  タップ→詳細ポップアップ。宝箱へ戻す/SNS共有。総展示スコア。枚数制限なし。
- 整理: 満杯時起動。入れ替え選択。確定/キャンセル。
- チュートリアル: Lv1実操作ガイド。設定から再生可。初回フラグ保存。

## 9. UI演出・サウンド（juice）
- 切る瞬間: スパッ音 + めくれ舞うパーティクル + スローモーション(80〜150ms) + 切り口フラッシュ。
- スコア: ドン表示(バウンス+シェイク)。品質スタンプ。Sランク全画面演出。
- 失敗: ビクッと変顔 + 「！」エフェクト。
- テーマカラー: 草原=緑/海=青/火山=赤/雷=黄、高彩度+グロー。
- シェア画像: ボード全体を1枚で。プレイヤー名/襟/総展示スコア。
- 縦持ち最適化、なぞり音/確定/スコア/失敗の専用SE、テーマ別BGM。

## 11. MVP範囲
- 画面: レベル選択/ゲーム/リザルト/失敗/宝箱/エリボード/整理/チュートリアル/設定
- Lv1〜4(+将来5)、スコア計算、本体接触判定、自己交差確定、レベル解放、宝箱保存(移動)、ボード展示、画像生成、SNS共有、ローカル保存
- ランダム変化は1種のみ（襟出現時の回転ランダム）
- 含まない: オンライン/アカウント/課金/マルチ/レア襟/サーバー保存 等
